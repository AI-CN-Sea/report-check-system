# 接口文档

## 1. 通用返回格式

```json
{
  "code": 200,
  "message": "success",
  "data": {}
}
```

## 2. 健康检查

```http
GET /api/health
```

返回示例：

```json
{
  "code": 200,
  "message": "success",
  "data": {
    "status": "UP",
    "time": "2026-05-14T17:30:00",
    "service": "report-check-backend"
  }
}
```

## 3. 后续接口规划

| 模块 | 方法 | 路径 | 说明 |
| --- | --- | --- | --- |
| 登录 | POST | /api/auth/login | 用户登录 |
| 注册 | POST | /api/auth/register | 教师/学生注册 |
| 找回密码 | POST | /api/auth/forgot-password | 根据用户名和邮箱重置密码 |
| 用户 | GET | /api/users | 用户列表 |
| 课程 | GET | /api/courses | 课程列表 |
| 班级 | GET | /api/classes | 班级列表 |
| 实验任务 | GET | /api/experiment-tasks | 实验任务列表 |
| 报告 | POST | /api/reports/upload | 上传报告 |
| 查重 | POST | /api/check-tasks | 创建查重任务 |
| 查重结果 | GET | /api/check-results | 查询查重结果 |
| 统计 | GET | /api/statistics/dashboard | 首页统计 |

## 4. 已实现接口

### 4.1 用户登录

```http
POST /api/auth/login
```

请求体：

```json
{
  "username": "teacher01",
  "password": "123456"
}
```

### 4.1.1 注册与找回密码

```http
POST /api/auth/register
POST /api/auth/forgot-password
```

注册只允许 `TEACHER` 和 `STUDENT`，管理员账号由后台创建。

### 4.2 用户列表

```http
GET /api/users
```

### 4.2.1 角色列表

```http
GET /api/users/roles
```

### 4.2.2 新增用户

```http
POST /api/users
```

请求体：

```json
{
  "username": "student04",
  "password": "123456",
  "realName": "测试学生",
  "roleId": 3,
  "studentNo": "20260014",
  "email": "student14@example.com",
  "status": 1
}
```

### 4.2.3 编辑用户、重置密码、删除用户

```http
PUT /api/users/{id}
PUT /api/users/{id}/password
DELETE /api/users/{id}
```

### 4.3 课程列表

```http
GET /api/courses
```

### 4.4 班级列表

```http
GET /api/classes
```

班级管理：

```http
POST /api/classes
```

```http
GET /api/student-classes
GET /api/student-classes?classId=1
```

### 4.5 实验任务列表

```http
GET /api/experiment-tasks
```

### 4.6 首页统计

```http
GET /api/statistics/dashboard
```

### 4.7 新增实验任务

```http
POST /api/experiment-tasks
```

请求体：

```json
{
  "courseId": 1,
  "classId": 1,
  "title": "实验二：查重系统实现",
  "description": "提交系统设计和核心代码说明",
  "deadline": "2026-06-20T23:59:59",
  "status": 1,
  "createdBy": 2
}
```

### 4.8 上传报告

```http
POST /api/reports/upload
```

表单参数：

| 参数 | 说明 |
| --- | --- |
| taskId | 实验任务 ID |
| studentId | 学生 ID |
| file | 报告文件，支持 txt、docx、pdf |

### 4.8.1 批量上传报告

```http
POST /api/reports/batch-upload
```

表单参数：

| 参数 | 说明 |
| --- | --- |
| taskId | 实验任务 ID |
| files | 多个报告文件 |

批量上传时，文件名需要包含学生姓名、用户名或学号，例如：

```text
李明-实验一.txt
student02-report.docx
20260003-report.pdf
```

### 4.9 报告列表

```http
GET /api/reports
GET /api/reports?taskId=1
```

### 4.9.1 报告详情

```http
GET /api/reports/{id}
```

说明：

- 教师可以查看所有报告解析文本。
- 学生只能查看自己的报告解析文本。
- 报告上传限制为 `txt`、`docx`、`pdf`，单文件不超过 20MB。
- 任务已截止或已归档时不允许提交。
- 上传学生必须属于实验任务对应班级，否则后端会返回中文原因提示。

### 4.10 发起查重

```http
POST /api/check-tasks
```

请求体：

```json
{
  "experimentTaskId": 1,
  "createdBy": 2
}
```

### 4.11 查重任务列表

```http
GET /api/check-tasks
```

### 4.12 查重结果列表

```http
GET /api/check-results
GET /api/check-results?checkTaskId=1
```

### 4.13 查重结果详情

```http
GET /api/check-results/{id}
```

### 4.14 导出查重结果

```http
GET /api/check-results/export
GET /api/check-results/export?checkTaskId=1
```

返回 CSV 文件，包含报告双方、余弦相似度、SimHash 相似度、综合相似度和风险等级。
