## 3. 演示闭环

1. 登录系统。
2. 进入首页看板，查看课程、任务、报告和查重任务数量。
3. 进入“实验任务”，新增或编辑实验任务。
4. 进入“报告上传”，选择实验任务和学生，上传 `.txt`、`.docx` 或 `.pdf` 报告。
5. 系统保存文件，并提取报告文本。
6. 进入“查重结果”，选择实验任务并点击“发起查重”。
7. 系统对同一实验任务下的报告两两比较。
8. 查看综合相似度、余弦相似度、SimHash 相似度和风险等级。
9. 点击“详情”，查看相似句子。
10. 进入“统计分析”，查看风险等级占比和核心数据统计图。

## 3.1 三类角色界面区别

| 角色 | 可见菜单 | 说明 |
| --- | --- | --- |
| 管理员 | 系统看板、用户管理、课程管理、班级管理、任务查看、统计分析 | 维护用户、课程、班级和班级学生关联 |
| 教师 | 首页看板、实验任务、报告上传、查重结果、统计分析 | 负责创建任务、上传或导入报告、发起查重、查看结果 |
| 学生 | 我的首页、实验任务、我的报告 | 只能查看任务并提交自己的报告，不能查看他人报告和查重结果 |

后端同样做了接口权限控制。仅隐藏前端菜单是不够的，所以系统会在后端解析 JWT 并按角色拦截接口。

## 4. 后端调用链路

```text
Controller -> Service -> Mapper -> MySQL
```

报告上传流程：

```text
ReportFileController.upload
  -> ReportFileService.upload
    -> 保存文件到 uploads/
    -> TextExtractUtil.extract
    -> 写入 report_file 表
```

查重流程：

```text
CheckTaskController.startCheck
  -> CheckTaskService.startCheck
    -> 查询同一实验任务下解析成功的报告
    -> SimilarityCalculator.calculate
    -> 写入 check_task 表
    -> 写入 check_result 表
    -> 写入 similar_sentence 表
```

## 5. 查重算法说明

系统先进行文本清洗、停用词过滤和特征提取，再计算 TF-IDF 余弦相似度和 SimHash 相似度。

```text
FinalScore = 0.7 × CosineSimilarity + 0.3 × SimHashSimilarity
```

| 综合相似度 | 风险等级 |
| --- | --- |
| >= 80% | 高风险 |
| 60%-80% | 中风险 |
| 40%-60% | 低风险 |
| < 40% | 正常 |

## 6. 样例报告

```text
samples/generated/task1-txt/    12 份 txt 报告
samples/generated/task2-docx/   12 份 docx 报告
```

这些报告覆盖高风险、中风险、低风险和正常文本等测试场景，高风险和中风险样例包含可匹配的相似自然句。

## 7. 上传限制

- 支持文件：`txt`、`docx`、`pdf`。
- 单个文件大小：不超过 20MB。
- 学生只能提交自己的报告。
- 教师只能为学生用户上传报告。
- 上传学生必须属于实验任务对应班级。
- 任务已截止或已归档时，不允许继续提交。
- 同一任务下同一学生重复上传时，系统会覆盖旧记录并清理旧文件。
