# 需求规格说明

## 1. 系统目标

实验报告智能查重与分析系统用于辅助教师对学生提交的实验报告进行批量查重、相似度分析和重复风险识别。

系统只提供相似度依据和疑似重复内容，不直接判定学生抄袭。

## 2. 用户角色

| 角色 | 说明 |
| --- | --- |
| 管理员 | 管理用户、角色、课程、班级和系统日志 |
| 教师 | 创建实验任务、上传报告、发起查重、查看结果 |
| 学生 | 查看任务、提交实验报告、查看提交状态 |

## 3. 核心功能

| 模块 | 功能 |
| --- | --- |
| 登录认证 | 用户登录、JWT 鉴权、角色识别 |
| 用户管理 | 用户新增、编辑、禁用、重置密码 |
| 课程班级 | 课程管理、班级管理、学生班级关联 |
| 实验任务 | 创建任务、设置截止时间、查看提交情况 |
| 报告上传 | 单文件上传、批量上传、文件校验 |
| 文本解析 | docx、txt、PDF 文本提取 |
| 查重算法 | TF-IDF、余弦相似度、SimHash、风险等级 |
| 查重结果 | 相似度排序、相似句子、报告对比 |
| 统计分析 | 风险占比、相似度分布、班级平均重复率 |

## 4. 非功能需求

- 系统应支持前后端分离。
- 上传文件应限制类型和大小。
- 后端接口应返回统一 JSON 格式。
- 数据库表结构应清晰，便于最终报告说明。
- 系统应能在本机稳定启动和演示。
