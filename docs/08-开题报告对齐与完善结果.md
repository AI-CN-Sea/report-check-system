# 开题报告对齐与系统完善结果

## 1. 总体结论

结合开题报告中“基于 Spring Boot 与文本相似度算法的实验报告查重系统设计与实现”的目标，当前 `report-check-system` 已经形成完整的业务闭环。系统能够围绕课程、班级、实验任务、学生报告和查重结果完成从任务创建、报告上传、文本解析、批量两两查重、风险等级判定到统计图表展示的主要流程，符合开题报告中对可运行 Web 应用系统的预期。

本次完善后，算法实现进一步向开题报告中的描述靠拢：查重任务在计算余弦相似度时，不再只依据当前两份文本构建 IDF，而是将同一实验任务下全部已解析报告作为语料集合，统一计算词项的文档频率和 IDF 权重。这样更符合“同一实验任务下多份报告统一构建词典和权重”的设计要求。

## 3. 本次代码完善内容

### 3.1 算法实现优化

修改文件：

```text
backend/src/main/java/com/reportcheck/algorithm/SimilarityCalculator.java
backend/src/main/java/com/reportcheck/service/CheckTaskService.java
```

完善点：

1. 新增 `calculateWithCorpus` 方法，支持传入同一实验任务下的全部报告文本作为语料集合。
2. 查重任务生成结果时，使用当前任务下所有解析成功报告统一计算 IDF 权重。
3. 保留原有 `calculate` 方法，便于单元测试和两文本独立比较。
4. 修正空文本场景下 SimHash 相似度误判问题。原逻辑中两个空文本的 SimHash 指纹相同，可能得到较高相似度；现在只要任一文本为空，SimHash 相似度直接返回 0。
5. 相似句匹配仍采用两句之间的局部 TF-IDF 余弦相似度，适合作为详情页的辅助解释信息。

这使系统更符合开题报告中的算法流程：

```text
读取同一实验任务下所有已解析报告
进行文本清洗、切分和停用词过滤
基于报告集合统计词项文档频率
计算 TF-IDF 权重向量
两两计算余弦相似度
同时计算 SimHash 指纹相似度
加权得到综合相似度
根据阈值划分风险等级
```

### 4.2 中文分词属于轻量级实现

当前系统没有接入 HanLP、Jieba 等第三方中文分词库，而是采用中文二元词条切分。论文中不要写成“基于成熟中文分词模型”，建议写成：

```text
系统采用轻量级中文二元词条切分方法，将中文连续文本转换为相邻二字词条，并结合停用词过滤完成文本预处理。该方法部署简单、计算成本低，适合课程设计场景下的实验报告近重复检测。
```

### 4.3 安全实现为基础级别

系统已经实现 JWT 登录认证和角色权限控制，但数据库中密码仍为明文。课程设计演示可以接受，但论文中不要强调“高安全性”。建议写：

```text

```

如果后续继续完善，可以引入 BCrypt，对初始化用户密码、登录校验和重置密码逻辑进行统一改造。

## 6. 建议写入论文的系统实现表述

可以在论文“系统实现”章节中使用如下表述：

```text
本系统采用前后端分离架构，前端基于 Vue 3、Element Plus 和 ECharts 实现后台管理界面，后端基于 Spring Boot、MyBatis-Plus 和 MySQL 实现业务接口、文件解析、查重计算和数据持久化。系统面向高校实验报告批量检查场景，围绕课程、班级、实验任务、报告文件和查重结果建立业务模型，支持管理员、教师和学生三类用户协同使用。

在查重算法方面，系统首先对上传的 txt、docx 和 pdf 报告进行文本提取，然后完成文本清洗、轻量级中文二元词条切分和停用词过滤。对于同一实验任务下的全部已解析报告，系统统一统计词项文档频率并计算 TF-IDF 权重，将报告文本转换为向量表示，再对任意两份报告计算余弦相似度。同时，系统使用 SimHash 方法生成文本指纹，通过汉明距离计算文本指纹相似度。最终系统按照 0.7 和 0.3 的权重融合余弦相似度与 SimHash 相似度，得到综合相似度，并依据阈值划分正常、低风险、中风险和高风险等级。

系统不直接判定学生是否抄袭，而是为教师提供辅助复核依据。教师可以查看相似报告组合、综合相似度、风险等级、相似句子和统计图表，从而快速定位需要重点检查的报告。
```

## 7. 后续优化路线

按优先级建议如下：

1. 固定演示数据库和样例报告，确保登录、上传、查重、查看结果、统计图表完整可用。
2. 优化查重结果页视觉效果，使高风险结果、综合相似度和相似句子更醒目。
3. 拆分前端 `DashboardView.vue`，提升代码结构清晰度。
4. 增加 BCrypt 密码加密，提升认证安全性。
5. 增加 Word/PDF 查重详情报告导出。
6. 增加操作日志模块，对登录、上传、查重等关键行为进行记录。
7. 如时间允许，接入 Jieba 或 HanLP，提升中文分词精度。
8. 增加历史报告库查重或跨任务查重能力。
