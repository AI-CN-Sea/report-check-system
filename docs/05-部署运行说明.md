# 部署运行说明

## 1. 环境要求

- JDK 17
- Maven 3.9.x
- Node.js 和 npm
- MySQL 8.0
- Navicat 16 或 17

## 2. 数据库初始化

1. 打开 Navicat。
2. 连接本机 MySQL。
3. 执行 `database/init.sql`。
4. 如需直接展示查重结果和相似句，继续执行 `database/demo_seed.sql`。
5. 确认生成 `report_check_system` 数据库。

## 3. 后端配置

后端默认读取本机 MySQL：

```yaml
spring:
  datasource:
    username: root
    password: root
```

如果换机器运行，可以直接修改 `backend/src/main/resources/application.yml`，也可以通过环境变量覆盖：

```powershell
$env:DB_URL="jdbc:mysql://localhost:3306/report_check_system?useUnicode=true&characterEncoding=utf8&serverTimezone=Asia/Shanghai&useSSL=false"
$env:DB_USERNAME="root"
$env:DB_PASSWORD="你的数据库密码"
$env:REPORT_CHECK_JWT_SECRET="change-this-secret-before-demo-32bytes-minimum"
```

## 4. 后端启动

启动命令：

```bash
cd backend
mvn spring-boot:run
```

健康检查：

```text
http://localhost:8080/api/health
```

## 5. 前端启动

```bash
cd frontend
npm install
npm run dev
```

访问：

```text
http://localhost:5173
```

## 6. 一键启动脚本

Windows 环境可以运行项目根目录下的 `start-dev.ps1`。脚本会优先使用系统已有的 `JAVA_HOME`、`MAVEN_HOME` 和 `NODE_HOME`，如果没有设置，则使用脚本中的默认路径。

如果换机器运行，需要确认以下路径可用：

```text
JAVA_HOME
MAVEN_HOME
NODE_HOME
```

停止服务可以运行：

```powershell
.\stop-dev.ps1
```
